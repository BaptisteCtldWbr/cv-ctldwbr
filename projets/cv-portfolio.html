CV front

J'ai voulu proposé mon CV en version Web, logique, j'aspire à être développeur web !
Pour cela, je suis parti de mon CV print, et de son identité visuelle. J'ai fait en sorte de mettre toutes les informations qui était sur le côté du CV dans une première partie, et le reste à la suite.

  • Le header du site est conçu comme un réel "à côté" du CV avec les informations principales, les détails
  • Puis, toujours en suivant les standards d'un CV, on retrouve mes expériences et mes formations.

J'ai également fait en sorte que tout puisse être "responsive", c'est à dire que cela s'adapte à l'écran.

CV/Portfolio back

Puis, je me suis aperçu qu'il manquait quelque chose : des preuves de mes compétences (c'est toujours mieux que des lignes de CV). J'ai donc inventé un truc tout nouveau : le ✨ Portfolio ✨. Il existe plein d'outils "no-code" pour en faire un soi-même, en quelques instant. Mais je suis apprenti développeur web, je vais donc le développé.

Initialement, c'était juste pour mes projets : après mes expériences, je voulais mettre une sorte de galerie, assez simple, de ce que j'avais pu réaliser. Mais, je me suis chauffé : plutôt que de le faire simplement en front (HTML - CSS), je l'ai développé de manière à ce qu'il soit "connecté" à une base de données (avec PHP et MySQL), qui contient mes projets. J'en ai également profité pour faire en sorte que TOUT le reste dépende de cette base de données : contacts, technologies maitrisées, langues, centres d'intérêts avec mes différentes expériences. Et parce que "jamais deux sans trois", j'ai rajouté un formulaire de contact par mail et base de données !

Portfolio

J'ai donc imaginé une table, contenant les informations de tous mes projets.

Puis, tout simplement j'affiche tous les projets, dans un conteneur grid, avec l'image du projet, son nom court, ses tags et son contexte (le petit émoji). Et j'ai aussi fait en sorte de pouvoir cacher certains projets, s'ils sont en cours de rédaction, ou juste nul...

Projets

C'est bien beau d'afficher plein de projets, mais encore faut-il faire les présentations ! Pour cela, chaque projet redirige vers sa propre page. Comme vous pouvez le remarquer, tout ce passe au niveau des paramètres d'URL, j'ai codé une page en PHP qui prend le paramètre d'URL, cherche dans la base de données quel projet a se nom, et l'affiche.

On commence avec une présentation rapide : nom, miniature, contexte et date du projet, tags et courte description, avec les liens permettant de voir les réalisations en question.

Et, ce que vous êtes en train de lire : le contenu du projet. Pour cela, rien de bien technique : dans la base de données, une chaîne de caractères qui pointe vers un fichier HTML, que j'inclus ensuite.

CV - BDD

J'avais déjà fait une base de données, donc autant en profiter : tout gérer avec ça ! Mes liens de contacts, les technologies ect...

J'ai également transformé le code statique de mes expériences en dynamique, évidemment c'est toujours géré par une base de données !

Formulaire de contact

J'en ai également profité pour développer un formulaire de contact. A la soumission, il m'envoie un mail (avec vous en copie) et enregistre le message dans une table dédiée de la base de données.
